# Yumi Tools

This directory is generated by `yumi --edge`.

If you are a coding agent working inside this generated project, read
`../AGENTS.md` first. It is copied from the canonical Yumi SDK agent guide and
explains how to wire the generated SDK into the host app without needing the
Yumi source tree.

Each language gets:

- a small setup file that you edit
- a local SDK copy that you usually do not edit
- a language-specific `README.md`

## Directory Layout

```text
yumi_tools/
├── .env
├── python/
├── swift/
├── typescript/
├── cpp/
├── ue5/
├── go/
├── java/
├── csharp/
├── rust/
├── kotlin/
├── dart/
└── README.md
```

## What To Do Next

1. Open the folder for the language you actually use
2. Read that folder's `README.md`
3. Edit the generated setup file
4. Call the generated init function from your app entry point

## Generate Again

```bash
yumi --edge
```

Generate only some languages (repeat `--lang` or use commas):

```bash
yumi --edge --lang python
yumi --edge --lang rust --lang python
yumi --edge --lang rust,python
yumi --edge --lang swift
yumi --edge --lang typescript
yumi --edge --lang cpp
yumi --edge --lang ue5
yumi --edge --lang go
yumi --edge --lang java
yumi --edge --lang csharp
yumi --edge --lang rust
yumi --edge --lang kotlin
yumi --edge --lang dart
```
