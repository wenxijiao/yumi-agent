# Kumi Tools

This directory is generated by `kumi --edge`.

If you are a coding agent working inside this generated project, read
`../AGENTS.md` first. It is copied from the canonical Kumi SDK agent guide and
explains how to wire the generated SDK into the host app without needing the
Kumi source tree.

Each language gets:

- a small setup file that you edit
- a local SDK copy that you usually do not edit
- a language-specific `README.md`

## Directory Layout

```text
kumi_tools/
├── .env
├── python/

├── go/
├── java/
├── csharp/
├── rust/
├── kotlin/
├── dart/
└── README.md
```

## What To Do Next

1. Open the folder for the language you actually use
2. Read that folder's `README.md`
3. Edit the generated setup file
4. Call the generated init function from your app entry point

## Generate Again

```bash
kumi --edge
```

Generate only some languages (repeat `--lang` or use commas):

```bash
kumi --edge --lang python
kumi --edge --lang rust --lang python
kumi --edge --lang rust,python
kumi --edge --lang swift
kumi --edge --lang typescript
kumi --edge --lang cpp
kumi --edge --lang ue5
kumi --edge --lang go
kumi --edge --lang java
kumi --edge --lang csharp
kumi --edge --lang rust
kumi --edge --lang kotlin
kumi --edge --lang dart
```
